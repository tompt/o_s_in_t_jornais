Manually setting up Pi WiFi using wpa_supplicant.conf

Autor(es):Sebastian Bjurbom

Data:2017-04-08 22:17:00+01:00

URL:http://www.raspberrypi-spy.co.uk/2017/04/manually-setting-up-pi-wifi-using-wpa_supplicant-conf/

Although you can configure WiFi using the graphical utility within the Raspbian Desktop this requires that you connect a keyboard, mouse and monitor to your Pi. It is sometimes useful to be able to do it before you’ve booted the Pi. This is especially useful when using the Pi Zero W where attaching a keyboard and mouse requires a USB hub. The following technique will allow you to take a fresh SD card, configure your WiFi settings and boot a Pi Zero without any other wires than a power cable. It should work for both the Pi Zero and Pi Zero W but it’s easier on the W as you don’t need to worry about a WiFi dongle. Step 1 – Create a blank text file Create a blank text file named “wpa_supplicant.conf”. Use a plain text editor rather than a Word Processor. If using Windows you need to make sure the text file uses Linux/Unix style line breaks. I use Notepad++ (it’s free!) and this is easy to do using “Edit” > “EOL Conversion” > “UNIX/OSX Format”. “UNIX” is then shown in the status bar. Insert the following content into the text file : country=us update_config=1 ctrl_interface=/var/run/wpa_supplicant network={ scan_ssid=1 ssid="MyNetworkSSID" psk="Pa55w0rd1234" } Double check the SSID and password. Both the SSID and password should be surrounded by quotes. The Country Code should be set the ISO/IEC alpha2 code for the country in which you are using your Pi. Common codes include : GB (United Kingdom) FR (France) DE (Germany) US (United States) SE (Sweden) Step 2 – Copy to SD Card Copy the file to the boot partition on your SD card. In Windows this is the only partition you will be able to see. It will already contain the following files : bootcode.bin loader.bin start.elf kernel.img cmdline.txt Step 3 – Eject, Insert and Boot Safely remove the SD card from your PC and insert into the Pi. Power up the Pi and once it has booted you should be connected to your WiFi network. You may be able to use your Router admin interface to list connected devices. Your Pi should appear in the list with an assigned IP address. Additional Thougths As Sebastian Bjurbom points out in the comments below you might want to take this opportunity to enable SSH as well. It is disabled by default but it is easy to enable by copying a blank text file named “ssh” to the boot partition. This can be done at the same time “wpa_supplicant.conf” is copied across. class="post-container cf">\n\t\n\t\t<div class="post-content-right">\n\t\t\t<div class="post-content description " itemprop="articleBody">\n\t\n\t\t\t\t\t\n\t\t\t\t\n\t\t<p>Although you can configure WiFi using the graphical utility within the Raspbian Desktop this requires that you connect a keyboard, mouse and monitor to your Pi. It is sometimes useful to be able to do it before you&#8217;ve booted the Pi. This is especially useful when using the Pi Zero W where attaching a keyboard and mouse requires a USB hub.</p>\n<p><span id="more-6490"></span></p>\n<p>The following technique will allow you to take a fresh SD card, configure your WiFi settings and boot a Pi Zero without any other wires than a power cable. It should work for both the Pi Zero and Pi Zero W but it&#8217;s easier on the W as you don&#8217;t need to worry about a WiFi dongle.</p>\n<h2>Step 1 &#8211; Create a blank text file</h2>\n<p>Create a blank text file named &#8220;wpa_supplicant.conf&#8221;. Use a plain text editor rather than a Word Processor.</p>\n<p>If using Windows you need to make sure the text file uses Linux/Unix style line breaks. I use Notepad++ (it&#8217;s free!) and this is easy to do using &#8220;Edit&#8221; &gt; &#8220;EOL Conversion&#8221; &gt; &#8220;UNIX/OSX Format&#8221;. &#8220;UNIX&#8221; is then shown in the status bar.</p>\n<p><img class="aligncenter size-full wp-image-6491" src="http://www.raspberrypi-spy.co.uk/wp-content/uploads/2017/04/notepad_plus_plus_unix_eol.png" alt="Notepad Plus Plus EOL Format" width="500" height="600" srcset="http://www.raspberrypi-spy.co.uk/wp-content/uploads/2017/04/notepad_plus_plus_unix_eol.png 500w, http://www.raspberrypi-spy.co.uk/wp-content/uploads/2017/04/notepad_plus_plus_unix_eol-250x300.png 250w" sizes="(max-width: 500px) 100vw, 500px" /></p>\n<p>Insert the following content into the text file :</p>\n<pre>country=us\r\nupdate_config=1\r\nctrl_interface=/var/run/wpa_supplicant\r\n\r\nnetwork={\r\n scan_ssid=1\r\n ssid="MyNetworkSSID"\r\n psk="Pa55w0rd1234"\r\n}</pre>\n<p>Double check the SSID and password. Both the SSID and password should be surrounded by quotes.</p>\n<p>The Country Code should be set the\xc2\xa0<a href="https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2#Officially_assigned_code_elements" target="_blank">ISO/IEC alpha2 code</a> for the country in which you are using your Pi. Common codes include :</p>\n<ul>\n<li>GB (United Kingdom)</li>\n<li>FR (France)</li>\n<li>DE (Germany)</li>\n<li>US (United States)</li>\n<li>SE (Sweden)</li>\n</ul>\n<h2>Step 2 &#8211; Copy to SD Card</h2>\n<p>Copy the file to the boot partition on your SD card. In Windows this is the only partition you will be able to see. It will already contain the following files :</p>\n<ul>\n<li>bootcode.bin</li>\n<li>loader.bin</li>\n<li>start.elf</li>\n<li>kernel.img</li>\n<li>cmdline.txt</li>\n</ul>\n<h2>Step 3 &#8211; Eject, Insert and Boot</h2>\n<p>Safely remove the SD card from your PC and insert into the Pi. Power up the Pi and once it has booted you should be connected to your WiFi network.</p>\n<p>You may be able to use your Router admin interface to list connected devices. Your Pi should appear in the list with an assigned IP address.</p>\n<h2>Additional\xc2\xa0Thougths</h2>\n<p>As\xc2\xa0Sebastian Bjurbom points out in the comments below you might want to take this opportunity to enable SSH as well. It is disabled by default but it is easy to enable by copying a blank text file named &#8220;ssh&#8221; to the boot partition. This can be done at the same time &#8220;wpa_supplicant.conf&#8221; is copied across.</p>\n<h2>Troubleshooting</h2>\n<p>If after waiting a few minutes your Pi is not connected to your WiFi consider the following points :</p>\n<ul>\n<li>Check wpa_supplicant.conf exists in the boot partition and the filename is correctly spelt</li>\n<li>Check the file contains the text listed
